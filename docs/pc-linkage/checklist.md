# OneAPI Android PC Linkage Checklist

## 功能一致性

- [ ] Android 登录/注册流程与 PC 当前版本一致。
- [ ] 非 Codex/Claude 模块不改变业务规则。
- [ ] Codex/Claude 模型列表包含 DeepSeek 和 XiaomiMIMO。
- [ ] 模型过滤结果与 PC 当前版本一致。
- [ ] 权限模式文案与 PC 当前版本一致。
- [ ] 执行日志状态文案与 PC 当前版本一致。

## 设备绑定

- [ ] 未绑定设备时，Codex/Claude 发送按钮不可用。
- [ ] 在线设备可绑定。
- [ ] 离线设备不可作为新任务目标。
- [ ] 解绑后不再向该设备创建 job。
- [ ] 切换设备后新会话使用新设备。

## 发送与执行

- [ ] Android 创建 job 后服务器返回 job id。
- [ ] PC/Mac 可 claim 指定设备 job。
- [ ] PC/Mac 执行日志能写入服务器。
- [ ] Android 能看到执行日志。
- [ ] Android 能看到 AI 回复。
- [ ] Android 能看到失败详情。
- [ ] Android 能看到已停止状态。

## 日志呈现

- [ ] 日志按时序线性显示。
- [ ] AI 意图日志不会被新日志覆盖。
- [ ] 子级日志不重复父级标题。
- [ ] 无内容日志不显示空白点。
- [ ] 不显示“执行具体内容”固定区块。
- [ ] 文件变更在对应日志事件下展示。

## 交互确认

- [ ] CLI 等待确认时 Android 出现确认条。
- [ ] 允许后桌面端继续执行。
- [ ] 拒绝后桌面端按拒绝结果处理。
- [ ] 确认已处理后不会重复弹出。
- [ ] 多设备只处理当前绑定设备的 pending interaction。

## 弱网与恢复

- [ ] 断网期间保留当前会话内容。
- [ ] 恢复网络后补拉缺失事件。
- [ ] 事件不会重复显示。
- [ ] 后台切回前台后状态正确。
- [ ] 设备心跳超时后 UI 显示离线。

## 发布检查

- [ ] Android lint 通过。
- [ ] Android 单元测试通过。
- [ ] Android 集成测试通过。
- [ ] PC 执行端联动测试通过。
- [ ] Mac 执行端联动测试通过。
- [ ] 生产服务器接口权限检查通过。
- [ ] 不输出 token、API key、用户本地路径等敏感信息到普通日志。
